                    Anon • 2 years ago • edited

                    Ahhhh, I remember the good old times when I created hacks for games like UT. We first used the underlying engine, Unreal Engine, to suit our needs. But anti-cheaters (community driven) were always one step behind us. Publishing something meant, it would be blocked a few days later. So we started using the windows API to inject our code into a foreign process. They then started using AV techniques, mapping the memory, searching for our "LoadLibrary" calls, or if a module was loaded etc.. Then we reversed the LoadLibrary functionality, and replicated it, so we didn't have to use the windows API to load our libs :)

                    Fun times, I tell you =D
